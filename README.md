# StudySquad

Сервис для автоматической транскрипции лекций и генерации квизов на основе материала. Полностью serverless архитектура на Yandex Cloud.

## Стек

| Сервис | Назначение | Стоимость |
|--------|-----------|-----------|
| GitHub Pages | Хостинг фронтенда | Бесплатно |
| Yandex Object Storage | Хранение аудиофайлов | Копейки |
| Yandex Cloud Functions | Бизнес-логика | Бесплатно до 1M вызовов |
| Yandex API Gateway | Роутинг запросов | Копейки |
| Yandex Lockbox | Хранение секретов | Бесплатно |
| PostgreSQL | База данных | ~600 руб/мес |
| Cloudflare Workers | Прокси для OpenAI | Бесплатно до 100k/день |
| OpenAI Whisper | Транскрипция аудио | $0.006 за минуту |
| Яндекс OAuth | Авторизация | Бесплатно |

## Cloud Functions

### `auth-yandex`
Авторизация пользователя через Яндекс ID.

**Триггер:** HTTP (GET `/auth/yandex`)

**Что делает:**
1. Принимает `code` от Яндекс OAuth
2. Обменивает код на токен Яндекса
3. Получает данные пользователя (email, имя, yandex_id)
4. Создаёт или обновляет запись пользователя в БД (upsert)
5. Возвращает собственный JWT токен

**Секреты:** `PG_DSN`, `JWT_SECRET`, `YC_CLIENT_ID`, `YC_CLIENT_SECRET`

---

### `presign`
Генерация presigned URL для загрузки файла напрямую в S3.

**Триггер:** HTTP (GET `/presign`)

**Что делает:**
1. Проверяет JWT токен пользователя
2. Принимает имя файла, название лекции, предмет, флаг группового доступа
3. Создаёт запись в таблице `transcriptions` со статусом `pending`
4. Генерирует presigned URL для PUT запроса в Object Storage
5. Возвращает `upload_url` и `record_id`

**Секреты:** `PG_DSN`, `JWT_SECRET`, `S3_KEY`, `S3_SECRET`, `S3_BUCKET`

---

### `transcribe`
Транскрибация аудиофайла через OpenAI Whisper.

**Триггер:** S3 триггер — срабатывает при создании объекта в бакете с префиксом `audio/`

**Что делает:**
1. Получает `bucket_id` и `object_id` из события S3
2. Обновляет статус транскрипции на `processing`
3. Скачивает аудиофайл из Object Storage во временную папку `/tmp`
4. Отправляет файл в OpenAI Whisper через Cloudflare Workers прокси
5. Сохраняет результат в БД, обновляет статус на `done`
6. При ошибке устанавливает статус `error`

**Секреты:** `PG_DSN`, `S3_KEY`, `S3_SECRET`, `S3_BUCKET`, `OPENAI_KEY`

**Таймаут:** 300 секунд  
**Память:** 512 МБ

---

### `get-transcriptions`
Получение списка транскрипций или одной транскрипции по ID.

**Триггер:** HTTP (GET `/transcriptions`, GET `/transcriptions/{id}`)

**Что делает:**
- Без ID — возвращает список всех транскрипций пользователя + групповые транскрипции его группы
- С ID — возвращает одну транскрипцию с текстом и метаданными

**Секреты:** `PG_DSN`, `JWT_SECRET`

---

### `get-subjects`
Получение списка предметов.

**Триггер:** HTTP (GET `/subjects`)

**Что делает:**
1. Проверяет JWT токен
2. Возвращает список всех предметов из таблицы `subjects`

Предметы общие для всех пользователей — администрируются вручную через БД.

**Секреты:** `PG_DSN`, `JWT_SECRET`

---

### `generate-quiz`
Генерация квиза на основе транскрипции через GPT-4o-mini.

**Триггер:** HTTP (POST `/quizzes`)

**Что делает:**
1. Проверяет JWT токен
2. Принимает `transcription_id` и `title`
3. Загружает текст транскрипции из БД
4. Отправляет запрос в GPT-4o-mini через Cloudflare Workers прокси
5. Просит сгенерировать 5 вопросов с вариантами ответов в формате JSON
6. Сохраняет квиз в таблице `quizzes`
7. Возвращает `quiz_id` и массив вопросов

**Секреты:** `PG_DSN`, `JWT_SECRET`, `OPENAI_KEY`

**Таймаут:** 60 секунд

---

### `get-quiz`
Получение квиза по ID.

**Триггер:** HTTP (GET `/quizzes/{id}`)

**Что делает:**
1. Проверяет JWT токен
2. Возвращает квиз с вопросами по ID

**Секреты:** `PG_DSN`, `JWT_SECRET`

---

### `get-quizzes`
Получение списка всех квизов пользователя.

**Триггер:** HTTP (GET `/quizzes`)

**Что делает:**
1. Проверяет JWT токен
2. Возвращает все квизы привязанные к транскрипциям пользователя

**Секреты:** `PG_DSN`, `JWT_SECRET`

## Схема БД

```sql
subjects        — предметы (общие для всех)
groups          — учебные группы
users           — пользователи (авторизация через Яндекс ID)
transcriptions  — транскрипции лекций
quizzes         — квизы с вопросами в JSONB
```

## Архитектура загрузки файла

```
Фронт генерирует UUID
  → GET /presign → Function presign
  → INSERT в БД (status=pending)
  → Возвращает presigned URL
  → Фронт делает PUT напрямую в S3
  → S3 триггер срабатывает
  → Function transcribe
  → Скачивает файл → Whisper → UPDATE в БД (status=done)
```

## Cloudflare Workers прокси

OpenAI блокирует запросы с российских IP. Cloudflare Worker работает на серверах в Европе/США и проксирует все запросы к `api.openai.com`. Cloud Functions обращаются к Worker вместо OpenAI напрямую.

URL прокси: `https://openai-proxy.wladkorostilev1.workers.dev/v1`
